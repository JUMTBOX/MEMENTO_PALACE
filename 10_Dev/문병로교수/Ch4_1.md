## Ch4_1. 정렬

**선택 정렬(Selection sort)**

1. 배열 `A[0...n-1]`에서 가장 큰 원소를 찾아 배열의 맨 끝 원소 `A[n-1]`과 자리를 바꾼다.
2. 이제 원소 `A[n-1]`을 관심 대상에서 제외하고 나머지 원소들 `A[0...n-2]`를 대상으로 `1번`작업을 반복하면 된다.

```text
selectionSort(A[], n):
  for last <- n-1 downto 1
    A[0...last] 중 가장 큰 수 A[K]를 찾는다.
    A[k] <-> A[last]
```

- 여기서 수행 시간을 좌우하는 작업은 두 수를 비교하는 작업이다. `두 수를 비교하는 작업을 몇 번 하느냐가 선택 정렬의 점근적 수행 시간 척도가 된다.`
- O(n^2)

**버블 정렬(Bubble sort)**

- 선택 정렬처럼 제일 큰 원소를 끝자리로 옮기는 작업을 반복한다. 다만, 제일 큰 원소를 오른쪽으로 옮기는 방법이 다르다.
- 선택 정렬이 가장 큰 수를 찾은 다음 맨 끝자리 수와 바꾸는 반면, 버블 정렬은 왼쪽부터 한 칸씩 이동하면서 이웃한 두 수를 비교하여 바꾼다.
- 이 부분 때문에 버블 정렬이 선택 정렬보다 느리다.

```text
bubbleSort(A[], n):
  for last <- n-1 downto 1
    for i <- 0 to last-1
      if(A[i] > A[i+1])
        A[i] <-> A[i+1]
        sorted <- FALSE
    if(sorted = TRUE) return
```

- 버블 정렬 역시 `두 수를 비교하는 작업이 시간을 좌우한다.`

**삽입 정렬(Insertion sort)**

- `선택 정렬/버블 정렬`과 정반대로 착상한 정렬이다.
- `선택 정렬/버블 정렬`은 '정렬되지 않은'배열의 크기를 n부터 시작하여 하나씩 '줄이는데', 삽입 정렬은 '이미 정렬된 배열'의 크기를 1에서 시작하여 하나씩 '늘린다.'
- 핵심 작업은 이미 정렬되어 있는 <i>`i`</i>개짜리 배열에 하나의 원소를 더하여 정렬된 <i>`i+1`</i>개짜리 배열을 만드는 과정이다.

```text
insertionSort(A[], n):
  for i <- 1 to n-1
    A[0...i]의 적합한 자리에 A[i]를 삽입한다.
```

```text
### 더 자세히
insertionSort(A[], n):
  for i <- 1 to n-1
    newItem <- A[i]
    for(j <- i-1; 0 <= j and newItem < A[j]; j--)
       A[j+1] <- A[j]
    A[j+1] <- newItem
```

```javascript
/**
 * @param {Array<number>} A
 * @param {number} n
 */
function s(A, n) {
  for (let i = 0; i < n; i++) {
    const newItem = A[i];
    let j = i - 1;
    while (0 <= j && newItem < A[j]) {
      A[j + 1] = A[j];
      j -= 1;
    }
    A[j + 1] = newItem;
  }
  return A;
}
```

- 삽입 정렬의 시간 복잡도는 O(n^2), 최선의 경우 O(n)
- 비효율적인 정렬 알고리즘군에 속하지만 배열이 거의 정렬되어 있는 상태로 입력되는 경우에는 가장 매력적인 알고리즘이다.
- 배열이 완전히 정렬된 채로 입력된다면 내부에 중첩된 for루프는 한번도 실행되지 않으므로 첫 번째 for루프를 순회할 때마다 상수 시간이 소요된다.
  - ref. `셸 정렬(Shell sort)`

```text
삽입 정렬은 앞에서 소개한 선택 정렬이나 버블 정렬보다는 빠르다. 10만 개의 원소를 임의로 생성해서 정렬해보면
삽입 정렬은 선택 정렬보다 2.5배 정도 빠르고, 버블 정렬보다 11배 정도 빠르다.
```
