## Ch3. 점화식과 알고리즘 복잡도 분석

```text
점화식과 점근적 표기법은 알고리즘의 수행 시간을 분석하는 핵심 도구
점근적 분석은 입력의 크기가 충분히 큰 경우에 대한 분석을 뜻한다.
재귀 알고리즘은 수행 시간을 점화식으로 나타낼 수 있다.
ch3에서는 점화식을 사용하여 재귀적 관점에서 분석할 수 있는 알고리즘의 복잡도를 계산하는 세 가지를 소개한다.
```

**점화식**
- 어떤 함수를 자신과 똑같은 함수를 이용해 나타내는 것 (`아버지가 아들을 보면서 "나랑 똑같다"라고 말하는 경우`)
  - ex) 피보나치 수열의 점화식 `f(n)=f(n-1)+f(n-2)`
- 점화식은 재귀 함수의 복잡도를 구하는 데 유용하다.
- 점화식은 쉽게 말해, `자기 자신보다 작은 문제를 이용해서 전체 비용을 표현한 식`

```java
int sum(int n) {
  if(n == 1) return 1;
  return n + sum(n-1);
}
```
- `sum(n)`의 실행 시간을 `T(n)`이라고 한다면, `T(n) = T(n-1) + 1` 이다.
- 왜냐하면 `sum(n)`은 내부에서 `sum(n-1)`을 한 번 호출하고, 자기 자신이 하는 일은 덧셈/비교 정동의 상수 작업이기 때문

```java
// # 이진 탐색 예시
int binarySearch(int[] arr, int target) {
  // 매번 절반만 남김
}
```
- 이진 탐색은 한 번 실행할 때마다 문제의 크기가 절반이 된다.
- `T(n) = T(n/2) + 1` 이 식의 의미는
```text
 크기 n짜리 문제를 푸는 시간 = 크기 n/2짜리 문제를 푸는 시간 + 현재 단계에서 비교하는 시간 
``` 

**`반복 대치`: 점화식을 더 작은 변수에 대한 점화식으로 대치하는 작업을 반복하면서 경계조건에 이를 때까지 전개하는 방법**
- 쉽게 말해, 점화식을 계속해서 펼쳐서 규칙을 찾는 방법이다.
- 이진 탐색 점화식을 예시로 보면
```text
T(n) = T(n/2) + 1
```
- 한 번 펼친다면
```text
T(n) = T(n/2) + 1
```
- 두 번 펼친다면
```text
T(n) = T(n/4) + 1 + 1
```
- 세 번 펼친다면
```text
T(n) = T(n/8) + 1 + 1 + 1
```
- k 번 펼친다면
```text
T(n) = T(n/2^k) + k
```
- 재귀가 끝나는 시점은 문제의 크기가 1이 될 때이다.
```text
n / 2^k = 1 이고 따라서 n = 2^k , k =  log2n
-> 최초식 T(n) = T(n / 2^k) + k
-> 구해진 k 값을 대입하면 T(n) = T(n / 2^(log2n)) + log2n
-> n의 값은 2^k라고 했으므로 k 값을 대입하면 n = 2^(log2n)이 된다.
-> 즉, T(n) = T(2^(log2n)/2^(log2n)) + log2n 이 되어 T(n) = T(1) + log2n 이 된다.
-> T(1)은 상수 시간이므로 제외하고  
최종적으로 표현하면 T(n) = O(log n)이 나오게 된다.
```

**`추정 후 증명`: 점화식의 점근적 복잡도를 추정한 후 이를 수학적 귀납법의 원리를 이용해 증명하는 방법**

**`마스터 정리`: 특정한 모양으로 표현되는 점화식의 복잡도를 복잡한 계산이나 증명 없이 바로 알 수 있는 방법**
