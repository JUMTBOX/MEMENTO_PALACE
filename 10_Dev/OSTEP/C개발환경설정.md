1. Msys2 설치
   - ```bash
       # 설치 이후 MSYS2 UCRT64 터미널에서 실행
       pacman -S --needed mingw-w64-ucrt-x86_64-gdb
     ```
2. Cygwin 설치
3. 시스템 환경 변수 Path에 `C:/cygwin/bin` 경로 추가
4. vscode `C/C++ Extension Pack` 확장 설치
5. 실행 할 소스 파일에서 `Crtl + Shift + P` -> `Tasks: Run Task` -> `활성 파일 실행 with Cygwin` 작업 선택

**MSYS2 (소프트웨어 배포 및 개발 환경 플랫폼)**

- Windows에서 리눅스 스타일의 개발 환경을 제공해주는 소프트웨어 패키지 관리자이자 터미널 환경
- 리눅스의 `apt`나 `pacman` 같은 명령어를 사용하여, 윈도우에 필요한 개발 도구(`GCC, MinGW 등`)를 쉽게 설치하고 업데이트할 때 사용

**MinGW-w64 (컴파일러 가이드/런타임 라이브러리)**

- 원래 리눅스/유닉스용으로 만들어진 GCC 컴파일러를 Windows(64비트 및 32비트) 환경에서 사용할 수 있도록 이식한 프로젝트
- C/C++ 코드가 Windows 운영체제 위에서 오류 없이 돌아갈 수 있도록 Windows 전용 기능(API)과 연결해주는 다리 역할

**GCC (컴파일러 소스코드 번역기)**

- GNU 프로젝트에서 만든 세계에서 가장 널리 쓰이는 오픈소스 C/C++ 컴파일러
- C/C++ 소스코드(.c, .cpp)를 컴퓨터가 이해하고 실행할 수 있는 기계어 실행 파일(.exe)로 직접 번역(컴파일)하는 데 사용

**CMake (빌드 시스템 생성기)**

- 소스코드가 복잡하고 파일이 많을 때, 이를 어떻게 빌드할지 관리해주는 프로젝트 관리 및 빌드 구성 도구
- 소스코드 파일이 수십, 수백 개일 때 어떤 순서로 컴파일할지 지시서(Makefile 등)를 자동으로 만들어 줌

**Cygwin**

- Windows 안에서 리눅스(Unix) 프로그램들을 그대로 가져다 쓸 수 있도록 만들어주는 가상 리눅스 환경
- 리눅스/유닉스 환경에서만 돌아가도록 짜인 복잡한 오픈소스 프로그램이나 서버용 프로그램을 윈도우로 포팅(이식)할 때 사용
- 윈도우 커맨드 창(CMD) 대신, 리눅스의 bash 쉘과 `ls, grep, awk, sed` 같은 강력한 리눅스 명령어들을 그대로 쓰고 싶을 때 사용
- 실행할 때 반드시 `cygwin1.dll`이라는 가상 레이어 파일이 주위에 같이 있어야만 돌아감
